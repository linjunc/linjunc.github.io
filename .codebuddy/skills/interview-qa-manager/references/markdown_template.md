# Markdown 格式模板

## 面试题标准格式

每道面试题应遵循以下 Markdown 格式：

```markdown
## {问题标题}

### 简要回答
{简洁的一句话回答或核心要点}

### 详细解答
{详细的解释说明，可以包含多个段落}

#### 要点一
{具体内容}

#### 要点二
{具体内容}

### 代码示例
\`\`\`javascript
// 示例代码
const example = () => {
  // ...
};
\`\`\`

### 注意事项
- 注意点1
- 注意点2

### 相关问题
- 相关问题1
- 相关问题2
```

## 格式规范

### 标题层级
- `#` 用于文件主标题（每个文件只有一个）
- `##` 用于每道面试题的问题标题
- `###` 用于问题下的子标题（简要回答、详细解答等）
- `####` 用于更细的要点

### 代码块
- 使用三个反引号包裹代码
- 指定语言类型：javascript, typescript, css, html, bash 等
- 代码应该完整可运行或清晰表达意图

### 列表
- 使用 `-` 或 `1.` 创建列表
- 保持缩进一致
- 列表项简洁明了

### 强调
- **重要概念** 使用粗体
- `代码关键字` 使用行内代码
- *次要强调* 使用斜体

## 示例：完整的面试题

```markdown
## 什么是闭包？

### 简要回答
闭包是指函数能够访问其词法作用域外部变量的特性，即使该函数在其词法作用域外执行。

### 详细解答
闭包是 JavaScript 中的一个重要概念，它允许函数访问并操作函数外部的变量。

#### 形成条件
1. 函数嵌套：一个函数内部定义另一个函数
2. 内部函数引用外部函数的变量
3. 外部函数返回内部函数（或以其他方式使内部函数在外部可用）

#### 特点
- **持久性**：闭包会保持对外部变量的引用，即使外部函数已经执行完毕
- **私有性**：可以创建私有变量和方法
- **内存占用**：需要注意内存泄漏问题

### 代码示例
\`\`\`javascript
// 基础闭包示例
function outer() {
  let count = 0;
  
  return function inner() {
    count++;
    console.log(count);
  };
}

const counter = outer();
counter(); // 1
counter(); // 2
counter(); // 3

// 实际应用：创建私有变量
function createCounter() {
  let count = 0;
  
  return {
    increment() { return ++count; },
    decrement() { return --count; },
    getCount() { return count; }
  };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.getCount());  // 1
console.log(counter.count);       // undefined（私有变量）
\`\`\`

### 应用场景
- 数据封装和私有变量
- 函数工厂和柯里化
- 回调函数和事件处理
- 模块化模式

### 注意事项
- 过度使用闭包可能导致内存泄漏
- 在循环中创建闭包需要特别注意变量捕获问题
- 及时清理不需要的闭包引用

### 相关问题
- 闭包和作用域链的关系是什么？
- 如何避免闭包导致的内存泄漏？
- 箭头函数的闭包行为有什么特殊之处？
```

## 内容质量要求

### 准确性
- 确保技术概念准确无误
- 代码示例经过验证可运行
- 避免过时的知识和废弃的 API

### 完整性
- 包含核心概念、原理、示例
- 覆盖常见使用场景
- 提供相关问题引导深入学习

### 可读性
- 使用清晰的语言表达
- 合理使用标题和列表组织内容
- 代码添加必要注释

### 实用性
- 关注面试中的考察重点
- 提供实际应用场景
- 包含最佳实践和注意事项
